rules · read these when overwhelmed

how this roadmap actually works

  • one main thing per week — not five.
  • ship something visible every week, even if tiny or ugly.
  • reference sites (mdn, html5andcss3, css-tricks) are support, not extra curricula.
  • build first, read second, watch third.
  • update this page to match reality, not the fantasy version.
  • if you miss a week, you don’t restart — you keep going.

definition of done (per week)

  • the concrete output exists and runs (local or deployed).
  • roadmap.html is updated to reflect what actually happened.
  • there’s at least one obsidian nightly report for the week.
  • you can explain what you built in plain english to a non-dev friend.
